# CLV900 變頻器 Modbus 協議摘要

**來源：** `doc/900系列通用变频器说明书-V900.24.01.16.C版本（6按键）.pdf`
**型號：** clvdrives 900-0007M1
**章節：** 第 6 章「通讯」（p.56~57）+ 5.2.8 F7 通讯参数（p.48~49）+ 5.3 监视参数（p.53~55）

## 通訊基本

- **協議：** Modbus-RTU（透過 USR-TCP232-304 透傳走 TCP）
- **支援功能碼：** `0x03` 讀取、`0x06` 寫單一暫存器
  - **不支援** `0x10`（多寫）、不支援 bit-level 讀寫
- **單次讀取上限：** 連續 12 個暫存器
- **CRC：** 標準 Modbus CRC16（init `0xFFFF`，poly `0xA001`，LSB first）
- **錯誤回應：** function code `| 0x80`（例如 `0x83` 表示 0x03 異常）

## 控制 / 設定暫存器（write `0x06`）

| 寄存器 | 功能 | 範圍 / 說明 |
|---|---|---|
| `0x0001` | 通訊設定頻率 | -10000~+10000，10000 = 100% F8-03 最大頻率，**負值反向** |
| `0x0002` | 控制命令 | 1=正轉 / 2=反轉 / 3=正點動 / 4=反點動 / 5=自由停機 / 6=減速停機 / 7=故障復位 |
| `0x0003` | 繼電器/DO 控制 | BIT0=Relay1 / BIT1=Relay2 / BIT2=DO1 / BIT3=DO2（需 F1-08=7） |
| `0x0004` | AO1 輸出 | 0~0x7FFF（0~100%，需 F1-28=6） |
| `0x0005` | AO2 輸出 | 0~0x7FFF |

> `0x0001` 寫入時可帶正負（int16 兩補數），用 FC 0x06 寫、FC 0x03 讀。

## 功能參數 Fx-yy（read `0x03` / write `0x06`）

**位址公式：** `reg = 0xF000 + group*0x100 + index`
- F0-00 = `0xF000`，F9-40 = `0xF928`
- 例：F7-01 = `0xF701`，F7-19 = `0xF713`

### 啟用 Modbus 控制（一次性）
| 參數 | 設定值 | 用途 |
|---|---|---|
| F0-00 | `2` | 運行命令給定方式 = 通訊控制 |
| F0-01 | `8` | 目標頻率給定方式 = 通訊設定 |
| F1-08 | `7` | 繼電器輸出 = 通訊控制（如要寫 reg 0x03） |
| F1-28 | `6` | AO 輸出 = 通訊控制（如要寫 reg 0x04/0x05） |

### F7 通訊參數
| 參數 | 預設 | 說明 |
|---|---|---|
| F7-00 | 1 | 本機地址（slave ID）；0 = 廣播 |
| F7-01 | 0 | 波特率 0:9600 / 1:19200 / 2:38400 / 3:57600 / 4:115200 |
| F7-02 | 3 | 資料格式 0:8N2 / 1:8E1 / 2:8O1 / 3:8N1 |
| F7-03 | 0.0 | 通訊超時秒數，0 = 不檢測；超時報 `Err16/Err18` |
| F7-19 | 0 | MODBUS 數據通訊格式 0=標準 / 1=非標準（**驅動假設 0**）|
| F7-20 | 0 | Modbus 參數兼容 0=無效 / 1=有效（向下相容旗標，預設即可）|

## 監視參數 U0-yy（**read `0x03` only**）

**位址公式：** `reg = 0x1000 + group*0x100 + index`

| 寄存器 | U-碼 | 名稱 | 單位 |
|---|---|---|---|
| `0x1000` | U0-00 | 運行狀態 | 1=正轉 / 2=反轉 / 3=停機 |
| `0x1001` | U0-01 | 故障碼 | 0 = 無故障 |
| `0x1002` | U0-02 | 設定頻率 | 0.1 Hz |
| `0x1003` | U0-03 | 運行頻率 | 0.1 Hz |
| `0x1004` | U0-04 | 運行轉速 | rpm |
| `0x1005` | U0-05 | 輸出電壓 | V |
| `0x1006` | U0-06 | 輸出電流 | 0.1 A |
| `0x1007` | U0-07 | 輸出功率 | 0.1 kW |
| `0x1008` | U0-08 | 母線電壓 | V |
| `0x1009` | U0-09 | 輸出轉矩 | 0.1 %（signed）|
| `0x100A` | U0-10 | 功率因數角度 | 0.1 ° |
| `0x100B` | U0-11 | DI 輸入狀態 | bitmask |
| `0x100C` | U0-12 | 繼電器/DO 輸出狀態 | bitmask |
| `0x101A` | U0-26 | IGBT 溫度 | °C |
| `0x101E` | U0-30 | 反饋速度真實值 | 0.1 Hz |
| `0x102A` | U0-42 | 產品序號 低 16 位 | — |
| `0x102B` | U0-43 | 產品序號 高 16 位 | — |

完整表 U0-00 ~ U0-71（`0x1000`~`0x1047`）見 PDF p.54~55。

## 故障碼（U0-01）

PDF 7.2 章列舉常見故障，例如：
- `Err02` 加速過流 / `Err03` 減速過流 / `Err04` 恆速過流
- `Err05` 加速過壓 / `Err06` 減速過壓 / `Err07` 恆速過壓
- `Err16` 通訊超時（單次間隔超過 F7-03）
- `Err18` 通訊故障

實際上機後遇到再對照 PDF p.58~62。

## 鋼索吊車（中間絞盤）使用建議

對應 `Crane_control_PI` 中間絞盤同步：

```cpp
CLV900_inverter inv;
inv.init(client, MIDDLE_WINCH_SLAVE);          // slave 7
inv.configureModbusControl();                   // 一次性，或用面板設好

// pay out 同步
inv.setFreqHz(20.0, 50.0);                      // 20 Hz @ 50 Hz max = 40%
inv.runForward();                                // 馬達轉向待實機確認

// 收繩
inv.setFreqHz(20.0, 50.0);
inv.runReverse();

// 正常停機
inv.stopDecel();                                 // 走 F3 減速曲線

// 緊急停機
inv.stopFree();                                  // 自由停（不走斜坡）

// 故障處理
uint16_t code;
if (!inv.readFaultCode(code) && code != 0) {
    // log + EVT inverter_fault code=<code>
    inv.resetFault();
}
```

**注意事項：**
1. **首次上電**用面板把以下設好，否則 Modbus 控不動：
   - F0-00 = 2、F0-01 = 8（運行 + 頻率走通訊）
   - F7-00 = 7（slave ID 對齊 motion_flow §2）
   - F7-01/02 對齊 RS485_crane 線上其他裝置 baud 與資料格式
   - F7-03 = 1.0~5.0（通訊超時，配合 RPi watchdog）
   - **F7-19 = 0**（標準 Modbus，驅動假設此值；若為 1 通訊格式不同，無法用本驅動）
   - F7-20 = 0（保持預設）
2. **方向**：`runForward/Reverse` 哪個方向放繩、哪個收繩，需實機確認；可在 Crane main 用宏定義包裝
3. **停機**正常用 `stopDecel`；watchdog/緊急用 `stopFree`
4. **同步策略 (C 案)**：washrobot 下 `pay_out N` → Crane 同時：
   - 啟動左右繼電器（ZS_DIO CH3/CH4）
   - 變頻器 `setFreqHz(...)` + `runForward()`（中間絞盤）
   - SD76 #1/#2/#3 計米回授到位 → `relay off` + `inv.stopDecel()`
